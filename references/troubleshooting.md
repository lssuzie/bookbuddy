# 错误处理与排查指南

在将 PDF 转换为有声书的整个工作流中，如果遇到异常，请参照以下表格进行排查：

| 错误现象 | 可能原因 | 解决办法 |
| :--- | :--- | :--- |
| **HTTPSConnectionPool ... ProxyError / Timeout** | 本地网络代理连接抖动或断开 | 检查代理软件状态。如果报错，直接重新运行 `generate_audio.py`，程序会自动跳过已生成的文件，执行断点续传。 |
| **API 返回 401 Unauthorized** | .env 文件里的 API Key 过期或格式有误 | 确认 API Key 格式是否有效，并确保 Base URL 指向正确的网关。 |
| **读音依然有错声调（如将“死”读成“四”）** | 文本中残存极其特殊的部首码点，或为多音字 | 1. 确认该段落是否确实经过了 NFKC 标准化。<br>2. 若是正常字但由于语义模型误读，可在 Markdown 文本中手动用同音字替换（如将“死”在特定错读语境临时写为拼音或用其他清晰同音字引导）。 |
| **ffmpeg 合并报错或生成的文件无法播放** | 某些分段 MP3 文件体积为 0 或由于连接超时导致写入损坏 | 在缓存目录中找到对应大小为 0 或极其微小（如 < 1KB）的分段文件，手动删除它，然后重新运行脚本，程序会自动重试并下载正确的音频。 |
