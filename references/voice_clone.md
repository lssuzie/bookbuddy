# 声音克隆 (VoiceClone) 高保真优化规范

在针对特定声音（如明星声线）进行 VoiceClone（零样本声音克隆）时，由于模型本身的自回归特征，容易发生“越读越慢、越读越沉、发音机械、断续”等问题。请务必遵循以下克隆规范：

## 1. 声学种子自我提纯 (Acoustic Seed Self-Purification)

* **痛点**：原始采访或录音素材往往不可避免地带有细微的背景音乐（BGM）、降噪过度引入的相位电声等。
* **方案**：先剪辑一段 5-8 秒语调最契合的音频进行首次合成。若效果神似但包含细微缺陷，可直接将**首次合成出、声学上 100% 纯净无噪音的优秀合成干声**作为后续全书克隆的 `reference_audio`。这种迭代能彻底消灭金属电音与杂音。

## 2. 黄金字数严格受限 (75-105 字)

* 单次合成字数严格控制在 **75-105 字**。
* 字数太少缺乏语流，字数太多（如超过 120 字）会导致注意力概率在长句后半段发散，使模型自适应变慢、喉音变沉。

## 3. 剔除冲突修饰词，释放原有特质

* 避开在 Prompt 中使用“字字分明”（会导致吐字呆板割裂、失去连贯性）、“语气要诚恳谦逊”（对自带磁性的音色会产生过度静音负反馈导致有气无力）等主观描述。
* 宜使用“用随性、平静、毫无修饰的自然语调进行朗读，像是在私底下和老朋友聊天，注入温和且生动的感情”等顺畅且富有画面感的自然指令。

## 4. 零人工标签自然标点停顿

* 彻底摒弃所有的 `<break>` 标签与中括号 `[停顿]` 控制符，这不仅使文本格式清爽，还能完全释放 TTS 模型基于中文标点（。？！）进行自然换气和顿挫的流畅拼读感。
