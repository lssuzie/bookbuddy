# BookBuddy · AI 读书伴侣 Skill

> 把读书变成愉悦、享受、沉浸的体验。

## 核心原则：一个声音，做所有事

BookBuddy 不是切换音色的工具。**你找到一个喜欢的声音，它陪你做所有事。**

读论文、生成播客、教英语、睡前讲故事——都是同一个声音。就像 Samantha 不会为你读诗用一个声音、聊天用另一个声音。

---

## 入口判断

收到用户输入后，先判断：

| 用户说 | 走哪条路 |
|:-------|:---------|
| "用梁朝伟读" / "Tony 帮我读" | **直接走梁朝伟分支**（见下方） |
| "帮我做个播客" / "帮我读一下"（没说用什么声音） | **先问"用梁朝伟还是女声？"** |
| "读这个文件"（没说用什么声音） | **先问"用梁朝伟还是女声？"** |
| 回答了声音问题（"我喜欢林志玲的声音"等） | **走下方第一步：找到你的声音** |
| "不知道有什么推荐" | **走下方第一步：找到你的声音** |

### 快捷入口：梁朝伟 vs 女声

如果用户没说用哪个声音，先问：

> **"用梁朝伟还是女声？"**

**如果用户选"女声"：**

直接用 BookBuddy 默认女声（温柔私语 👑 温暖磁性的年轻女声）：

```bash
python3 <skill>/scripts/generate_audio.py <输入文件> \
  --voice-design "温柔私语" \
  --design-prompt "用明快有节奏的语气朗读，像是在和朋友聊天分享" \
  -s 1.15 \
  -o <输出文件>
```

> 如果用户之后说"换一个"，再走下方第一步的完整流程。

**如果用户选"梁朝伟"：**

用 tony-leung-tts 的声音克隆：

```bash
REF_AUDIO=$(find ~/.agents/skills/tony-leung-tts/梁朝伟_普通话参考.mp3)
python3 <skill>/scripts/generate_audio.py <输入文件> \
  --voice-clone --ref-audio "$REF_AUDIO" \
  --clone-prompt 标准流利 \
  -o <输出文件>
```

> 如果找不到参考音频，告诉用户先安装 tony-leung-tts skill。

## 第一步：找到你的声音

用户来了，第一句话：

> **"你有自己喜欢的声音吗？"**

### 特殊处理：用户直接说想听什么

如果用户不回答声音问题，直接说"帮我读道德经"、"读个故事给我听"、"睡前想听点东西"等——**不要强行让用户选声音**。

退而求其次，**根据内容类型推荐合适的声音**：

| 用户说 | 内容类型 | 推荐声音 |
|:-------|:---------|:---------|
| "帮我读这本书" / "读道德经" | 📚 知识阅读 | `知识讲述` 或 `白桦`（沉稳） |
| "读个故事给我听" | 📖 小说/故事 | `故事演绎`（有戏剧张力） |
| "Karpathy 做成播客" | 🎙️ 播客 | `播客主持`（自然聊天感） |
| "帮我学英语" | 🗣️ 学英语 | `Mia`（美式）或 `Chloe`（英式） |
| "睡前想听点东西" | 😴 睡前 | `睡前陪伴`（温柔缓慢） |
| "带我做个冥想" | 🧘 冥想 | `冥想引导`（空灵安宁） |
| "读首诗给我听" | 🫂 陪伴 | `温柔私语`（温暖磁性） |

用户不满意再说"换一个"。

```bash
# 按内容推荐，不满意再换
python3 <skill>/scripts/generate_audio.py 道德经.md --voice-design "知识讲述"
```

### 回答一：有，类似某个明星的风格

用户说"我想用林志玲的声音帮我读书"。

**两个方案：**

#### 方案 A（推荐 ✅ 安全）：用声音设计调一个类似风格

不提明星名字，把风格翻译成声音描述：

| 用户说 | 你翻译成 |
|:-------|:---------|
| "林志玲" | 温暖甜美有气质的女声 |
| "梁朝伟" | 低沉磁性有故事感的男声 |
| "Taylor Swift" | 年轻有活力的美式女声 |

然后直接调：

```bash
python3 <skill>/scripts/generate_audio.py <文件> --voice-design "温暖甜美有气质的女声，像知性偶像在娓娓道来"
```

**用户听到结果通常就满意了**——他在乎的是"好听"，不是"这是林志玲"。

不满意可以微调："快一点还是慢一点？更活泼还是更沉稳？"

#### 方案 B（仅当用户坚持）：用户提供参考音频做声音克隆

用户说"我就是想要她本人的声音"，可以提供一段参考音频。

**但需要注意风险**：
- 克隆真人声音需要本人授权
- 用于个人阅读（不公开传播）风险较低
- 用于商业用途会有侵权风险

```bash
python3 <skill>/scripts/generate_audio.py <文件> --voice-clone --ref-audio 参考音频.mp3
```

**推荐做法**：先试方案 A（声音设计），不满意再考虑方案 B。大多数用户试完方案 A 就满意了。

### 回答二：有参考音频

用户可以录 5 秒自己的声音，或者朋友的声音。

```bash
python3 <skill>/scripts/generate_audio.py <文件> --voice-clone --ref-audio 参考.mp3
```

### 回答三：不知道/想试试

让用户试听内置音色或声音设计预设：

```bash
# 内置音色（零准备）
冰糖（清亮甜美） / 茉莉（温柔知性） / 苏打（清爽活力） / 白桦（沉稳磁性）
Mia（美式女声） / Chloe（英式女声）

# 声音设计预设
温柔私语 👑  — BookBuddy 灵魂声线，温暖磁性，自然亲切
知识讲述 — 沉稳清晰，像老师在娓娓道来
故事演绎 — 有戏剧张力，能演绎情绪
播客主持 — 自然亲切，像朋友在聊天
睡前陪伴 — 温柔缓慢，越来越轻
冥想引导 — 空灵安宁，像从远处传来
```

推荐先试 **温柔私语**（BookBuddy 默认灵魂声线），不满意再换。

## 第二步：同一个声音，不同演绎方式

声音选好了，但读论文和读睡前故事的读法不一样。**声音不变，演绎方式变。**

| 场景 | 语速 | 语调 | 演绎方式 |
|:-----|:-----|:-----|:---------|
| 📚 读论文 | 正常 | 沉稳清晰 | 像认真在给你讲知识 |
| 📖 读故事 | 偏慢 | 有感情起伏 | 像沉浸在故事里 |
| 🎙️ 播客 | 偏快 | 自然聊天感 | 像朋友在分享 |
| 🗣️ 学英语 | 偏慢 | 清晰饱满 | 像在教你 |
| 😴 睡前 | 慢速 | 越来越轻 | 像在哄你入睡 |

用 `-s` 调语速，用 `--design-prompt` 调演绎方式：

```bash
# 同一个声音，不同读法
python3 <skill>/scripts/generate_audio.py 论文.md --voice-design "温柔私语" -s 1.0
python3 <skill>/scripts/generate_audio.py 故事.md --voice-design "温柔私语" -s 0.9
python3 <skill>/scripts/generate_audio.py podcast.md --voice-design "温柔私语" -s 1.1
python3 <skill>/scripts/generate_audio.py 睡前.md --voice-design "温柔私语" -s 0.7
```

如果内置的演绎方式不够，可以用 `--design-prompt` 自定义风格描述：

```bash
python3 <skill>/scripts/generate_audio.py 文件.md --voice-design "温柔私语" --design-prompt "用极其沉稳、有专业感的语气朗读，像是在做学术报告"
```

这样既保持了声音的一致性，又适应了不同场景。

## 第三步：用它做所有事

用户有了喜欢的声音，现在用它来做所有事。**不需要再问"用什么声音"了。**

### 📚 读一篇文字/一本书

用户有文件（论文、文章、小说、稿子），想转成音频。

**流程：**
1. 确认文件路径
2. 用已选好的声音调用 TTS
3. 返回音频

```bash
python3 <skill>/scripts/generate_audio.py <文件> --voice-design "温柔私语"
# 或
python3 <skill>/scripts/generate_audio.py <文件> --voice 白桦
# 或
python3 <skill>/scripts/generate_audio.py <文件> --voice-clone --ref-audio 我的声音.mp3
```

### 🎙️ 帮我做成播客

用户说一个话题（"Karpathy 最近说了什么"），没有现成文件。

**流程：**
1. **搜索**相关内容
2. **口语化总结**：300-500 字，短句为主，像朋友聊天
3. **保存为文本文件**
4. **用已选好的声音**调用 TTS
5. 返回音频

```bash
# 先把内容总结成播客文本 → 保存为 podcast.md
# 再用已选好的声音
python3 <skill>/scripts/generate_audio.py podcast.md --voice-design "温柔私语"
```

### 🗣️ 学英语

用户想听英文内容学习。

**两种方式：**
1. **用你选好的声音读英文**（同一个声音陪着你学英语）
2. 如果想听母语者发音：`Mia`（美式）或 `Chloe`（英式）

推荐方式 1——你喜欢的声音读英文，你更愿意多听。

```bash
python3 <skill>/scripts/generate_audio.py <英文文件> --voice-design "温柔私语"
```

### 😴 放松陪伴（睡前 / 冥想 / 陪伴）

用户不是来获取信息的，是想让声音陪着。

**用同一个声音**，只是语速不同：
- 睡前 → 慢速
- 冥想 → 极慢速
- 陪伴 → 正常语速

```bash
python3 <skill>/scripts/generate_audio.py <文件> --voice-design "温柔私语" -s 0.8  # 睡前慢速
```
