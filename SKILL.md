# BookBuddy · AI 读书伴侣 Skill

> **把读书变成愉悦、享受、沉浸的体验。**

**这是一个 Skill**，让你的 AI Agent 具备文本转语音能力。

## Skill 是什么？

Skill 是包含指令、脚本和资源的文件夹，Agent 动态加载以提升特定任务能力。BookBuddy Skill 教会 Agent 如何：
- 把文本转换为有声书
- 使用不同音色朗读
- 克隆声音
- 生成播客

## 安装方式

### Claude Code
```bash
/plugin install bookbuddy
```

### Claude.ai
直接在平台设置中上传 Skill 文件夹。

### Claude API
通过 Skills API 上传自定义技能。

### Coze / Dify / GPTs
参考 `coze-plugin/DEPLOY.md` 部署指南。

## 使用方式

安装后，你只需要对 Agent 说：
- "帮我读这本书"
- "用睡前催眠的声音读"
- "克隆我的声音"

Agent 会自动调用 BookBuddy Skill 完成。

## 核心能力

| 模式 | 说明 | 调用方式 |
|:-----|:-----|:---------|
| **基本 TTS** | 8 种内置音色 | `generate_audio --voice <音色名>` |
| **声音设计** | 6 种预设 + 自定义 | `generate_audio --voice-design "描述"` |
| **声音克隆** | 5 秒音频克隆 | `generate_audio --voice-clone --ref-audio <音频>` |
| **AI 代读 + 播客** | 总结 + 朗读 | 先总结 → 再调用 TTS |

## 使用场景

### 场景 1: 有声书（直接朗读）

用户说："帮我读这本书"、"生成有声书"、"把这本书转成音频"

**Agent 处理流程：**

1. **确认文件**：确认用户提供的文本文件
2. **询问声音偏好**（可选）：
   - 用户有指定 → 用指定声音
   - 用户无指定 → 推荐 `Her 知性元气` 或 `白桦`
   - 用户想克隆 → 引导录 5 秒音频
3. **调用 TTS**：
   ```bash
   python3 <skill-path>/scripts/generate_audio.py <文件> --clean
   # 或指定声音：--voice-design "Her 知性元气"
   # 或克隆：--voice-clone --ref-audio <音频>
   ```
4. **返回结果**：告知音频路径和时长

### 场景 2: 声音设计（创建独特声线）

用户说："用睡前催眠的声音读"、"用温柔叙述的声音"

**Agent 处理流程：**

1. 确认预设名称（睡前催眠/冥想引导/深夜电台/温柔叙述/Her 知性元气/悬疑小说）
2. 调用 TTS：`--voice-design "预设名"`
3. 返回结果

### 场景 3: 声音克隆（复刻声音）

用户说："用我的声音读"、"克隆这个声音"

**Agent 处理流程：**

1. 引导用户上传 5 秒参考音频
2. 调用 TTS：`--voice-clone --ref-audio <音频>`
3. 返回结果

### 场景 4: AI 代读 + 播客

用户说："帮我总结一下这本书"、"生成播客"、"读一下这个文档集"

**Agent 处理流程：**

1. **读完全文**：逐篇/逐章阅读用户给的文件
2. **口语化总结**：用讲故事的方式，像朋友聊天一样讲给用户听
   - 先讲故事本身（发生了什么）
   - 再给评价（好不好、值不值得读）
   - 每篇 300-500 字，口语化，短句为主
3. **生成播客文本**：保存为 Markdown
4. **调用 TTS**：同上（场景 1 步骤 3）
5. **告知结果**：音频路径 + 时长

**完成标准：**
- ✅ 播客文本已生成（口语化，非分析报告）
- ✅ 音频已生成
- ✅ 告知用户路径和时长
